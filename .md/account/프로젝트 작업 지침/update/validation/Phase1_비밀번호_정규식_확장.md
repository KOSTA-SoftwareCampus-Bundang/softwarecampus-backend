# 비밀번호 유효성 검사 규칙 업데이트 (Validation Update)

## 1. 개요
사용자 편의성을 높이기 위해 비밀번호에 허용되는 특수문자의 범위를 확장합니다. 기존에는 제한된 특수문자(`@$!%*#?&`)만 허용되었으나, 키보드에서 입력 가능한 대부분의 표준 특수문자를 허용하도록 변경합니다.

## 2. 변경 사항

### 2.1 기존 규칙 (AS-IS)
- **허용 특수문자**: `@ $ ! % * # ? &` (8종)
- **정규식**:
  ```regex
  ^(?=.*[A-Za-z])(?=.*\d)(?=.*[@$!%*#?&])[A-Za-z\d@$!%*#?&]{8,20}$
  ```

### 2.2 변경 규칙 (TO-BE)
- **허용 특수문자**: `! @ # $ % ^ & * ( ) - _ = + [ ] { } | ; : ' " , . < > / ? ~` (32종)
- **정규식**:
  ```regex
  ^(?=.*[A-Za-z])(?=.*\d)(?=.*[!@#$%^&*()\-_=+[\]{}|;:'",.<>/?~])[A-Za-z\d!@#$%^&*()\-_=+[\]{}|;:'",.<>/?~]{8,20}$
  ```
- **Java String Literal (Escaped)**:
  ```java
  "^(?=.*[A-Za-z])(?=.*\\d)(?=.*[!@#$%^&*()\\-_=+\\[\\]{}|;:'\",.<>/?~])[A-Za-z\\d!@#$%^&*()\\-_=+\\[\\]{}|;:'\",.<>/?~]{8,20}$"
  ```

## 3. 영향 범위 (Impact Analysis)

### 3.1 Backend
- **DTO**: `SignupRequest.java`
  - `@Pattern` 어노테이션의 `regexp` 값 수정 필요
- **테스트 코드**: `SignupRequestTest.java` (존재 시)
  - 다양한 특수문자를 포함한 비밀번호 테스트 케이스 추가 필요

### 3.2 Frontend
- **Validation Utils**: `src/utils/validation.ts`
  - `PASSWORD_REGEX` 상수 업데이트 (이미 반영됨)
- **문서**: `docs/api/01_signup.md`
  - API 명세서 내 비밀번호 제약조건 설명 업데이트 (이미 반영됨)
